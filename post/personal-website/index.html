    Blog built with vanilla HTML, CSS, and JS.

    Why I Built This Wesbite?

    Share Ideas and Showcase Projects

    The main reason I built this website is to share my ideas, creations, and findings on topics that I find interesting. I want to focus on making tutorials about software development, and I want to write articles about computer science, philosophy, and digital illustration. In addition, I want to show my software projects and document them on this website, so people can better understand my projects and be inspired to create new softwares. Software projects that I have created can be seen on the Projects page.

    Learn Building and Hosting a Website

    Building this website has taught me many things about web development. I learned how to make a maintainable web project so that it is easier to make changes and add features in the future. I have learned how to use the MDN Web Docs as reference when developing a website. Equally important is that I challenge myself to not use a web framework because I want to understand how to build user interface components, arrange layouts and style elements using CSS, and add behaviors to the elements using JavaScript. Next, I hosted this site on GitHub Pages because I can store my code and deploy it on the same platform. I can also see the changes on my website after I push my code to GitHub.

    What Tools and Methods Did I Use?

    I used Visual Studio Code as my code editor. VS Code has been a blessing for me because of the editing features, such as multi-cursor, Git integration, search feature, side by side editing feature, and the Live Server plugin. I used Mozilla Firefox as my browser and GitHub to save my code online.

    To manage HTML elements and CSS classes, I used the BEM (Block, Element, Modifier) Methodology made by Yandex. A block is a component that can be reused on different pages. An element is a part of a block that cannot be used independently. A modifier is a variant, such as style, state, or behavior, of a block or an element. I followed the BEM approach because it is the first method that I found, and it provides clear instructions on how to organize HTML elements and CSS rules. In addition, BEM documentation provides many examples and cases that make it easier to understand and implement. I can also inspect the BEM website to learn how the developers use the method.

    How Did I Build This Website?

    Inspect Element and Study Patterns

    I used Material UI Documentation and MUI Blog as my main references. As a starting point to style my website, I used MUI's dark mode style, emulated MUI components, and modified the components according to my needs. For the heading tree sidebar on post detail page, I was inspired by the sidebar on MDN Web Docs and Firefox PDF Reader. Hero title on the home page is inspired by the title on BEM Quick Start . I got my SVG icons from Font Awesome and GitHub Logos

    Follow Tutorials and Working Solutions

    To solve web development problems, such as how to toggle an HTML element I used W3Schools and Stack Overflow . The database design of this website was inspired by the Wordpress Database Description

    Read Documentation and Experiment

    My main documentation for HTML, CSS, and JS is the MDN Web Docs . I also read the Material UI Documentation for deciding component names and understanding a component's structure, style, and behaviors.
